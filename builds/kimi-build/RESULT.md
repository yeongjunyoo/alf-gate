# RESULT

## 무엇을 만들었나

채널톡 ALF 테스트(6/30 출시)의 "빈 시험지" 문제를 푸는 세팅 게이트.
과거 문의 로그와 상품 데이터에서 테스트 케이스, 성공 기준, 페르소나를 자동 생성하고,
약속 감사 4축(정책 날조, 발송상태 허위, 과잉 확약, 권한 밖 실행)으로 시뮬레이션을 채점한 뒤,
지식 변경 시 영향 케이스를 자동 재실행해 배포 보류를 권고한다.

## 실행법

```
python3 builds/kimi-build/run.py
```

한 방에 합성 데이터 생성 → 케이스 합성 → 시뮬레이션 → 약속 감사 → 리그레션 재실행 → `report.html` 생성까지 완주한다.
외부 의존은 `claude` CLI뿐이며 파이썬 stdlib만 사용한다.

## 파이프라인 구성

- `data.py` — 합성 데이터(카페24 상품 CSV 열, 채널톡 상담 CSV 열, 지식 아티클 3개, 태스크 승인 정책)와 케이스 자동 합성
- `engine.py` — `claude -p --max-turns 1 --model haiku` 서브프로세스 시뮬레이터, 결정론 룰 채점 + LLM judge(CX 점수, 보조 위반)
- `report.py` — 단일 파일 한국어 HTML 리포트(외부 CDN 없음): 게이트 판정 배너, KPI, 커버리지 맵, 약속 감사 4축, 케이스별 대화와 위반 증거 하이라이트, 리그레션 전후 비교
- `run.py` — 오케스트레이션과 게이트 판정
- 산출물: `report.html`, `out.json`, `stats.json`, `data/`

## 시연 3장면 대응

1. 케이스 자동 생성: 로그 30건을 유형별로 집계해 케이스 6개 합성, 커버리지 맵과 "다음 세팅 후보" 제시. 케이스는 ALF 테스트 포맷(목표, 성공 기준, 페르소나)과 필드 호환.
2. 약속 감사: 해결된 케이스에서도 감사 축 위반 2건 감지("CX 점수로는 통과인데 감사 축 실패"). 위반 문장을 대화에서 하이라이트해 증거로 제시.
3. 리그레션 게이트: 아티클 개정(반품 수치 삭제) → 영향 케이스 2개 자동 재실행 → C1 미해결 전환과 Δ -17%p 감지 → 배포 보류 권고 → 사람 승인.

## 정직한 한계

- 케이스 합성은 LLM이 아니라 유형별 템플릿 매칭이다(타임박스). 로그 → 케이스 문구 생성 자체를 LLM화하는 것이 다음 단계.
- 채점의 결정론 룰은 합성 데이터의 수치에 맞춰져 있어, 실제 로그 적용에는 룰 일반화가 필요하다.
- LLM judge는 haiku 단발 호출이라 판정이 실행마다 흔들릴 수 있다. 권위는 결정론 룰이 갖는다.
